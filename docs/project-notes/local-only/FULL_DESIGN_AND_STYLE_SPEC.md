# LESSON GENERATOR - FULL DESIGN AND STYLE SPEC

Updated: 2026-03-06

## 1. Purpose

This document defines the full visual and stylistic direction for the current Lesson Generator app.

It captures the requested design language for the app as it exists now and as it should continue to evolve. It is intended to guide future UI work, shared component decisions, page polish, wording tone, and visual consistency.

This spec is the working style authority for the streamlined wizard app, not for any older dashboard-style app architecture.

## 2. Core product identity

Lesson Generator is:

- teacher-first
- calm
- tactile
- trustworthy
- organized
- polished
- warm
- practical
- elegant
- classroom-real

Lesson Generator is not:

- a sterile SaaS dashboard
- a corporate admin panel
- a loud edtech toy
- a chaotic scrapbook
- a childish themed product
- a cluttered multi-color interface

The app should feel like:

- a beautiful teacher desk
- a calm planning workspace
- a lightly textured paper system
- a structured instructional engine
- a classy scrapbook with different textures, not riff raff

## 3. Emotional target

The app should make the teacher feel:

- grounded
- supported
- confident
- clear about next steps
- gently guided, not bossed around
- like the tool understands real classroom planning

The emotional tone should be:

- storybook warmth
- teacher-smart competence
- subtle craft and texture
- soft order and clarity
- polished without being fussy

## 4. Official visual direction

Primary direction:

APPLE ORCHARD STORYBOOK

Additional requested nuance:

CLASSY SCRAPBOOK WITH DIFFERENT TEXTURES

This means the visual language should combine:

- orchard warmth
- paper and canvas texture
- layered surfaces
- subtle crafted details
- restrained decorative accents
- visual softness
- clean hierarchy

This does NOT mean:

- junk journal clutter
- random stickers everywhere
- loud patchwork collage
- messy hand-made chaos
- too many patterns at once

The scrapbook idea must be curated, elegant, restrained, and teacher-friendly.

## 5. Product truths that style must support

The design should reinforce these truths:

- the app is for a real teacher doing real lesson planning
- the workflow matters more than decoration
- the UI should reduce uncertainty
- the app should show what the system used and why
- the system should feel trustworthy
- the system should feel usable in everyday classroom prep

For kindergarten and early elementary lesson generation, the product should visually support teacher-led instruction, not self-guided student navigation.

## 7. Typography

Preferred heading fonts:
- Playfair Display
- Libre Baskerville

Preferred body fonts:
- Inter
- Source Sans 3

Optional warmth accent font:
- Caveat

### Typography intent

Headings should feel:
- elegant
- grounded
- literary
- slightly storybook
- never fussy or wedding-invitation formal

Body text should feel:
- clean
- easy to scan
- professional
- classroom-practical
- readable at normal working sizes

Caveat should be used:
- rarely
- only for tiny warmth accents
- maybe a note label or subtle decorative caption
- never for core UI text
- never for dense content

### Typographic hierarchy

Hero titles:
- serif
- large
- warm
- strong visual presence

Section titles:
- serif or high-weight body font depending on context
- clear
- structured
- slightly more formal than body text

Labels:
- sans serif
- bold
- compact
- practical

Body copy:
- sans serif
- calm line spacing
- moderate contrast
- readable on cream and white surfaces

Helper text:
- muted
- slightly smaller
- still legible
- never too faint

### Typography avoid rules

- no tiny low-contrast body text
- no all-caps everywhere
- no overly playful classroom fonts
- no script font for real form fields or core controls
- no dense wall-of-text layout without spacing

## 8. Texture and surface language

Texture is important.

Requested texture mood:
- canvas
- paper
- tactile
- lightly painted
- scrapbook layers
- soft handmade quality

Texture should be:
- subtle
- barely-there
- elegant
- supportive
- not noisy

Good texture ideas:
- soft paper grain
- faint canvas wash
- very light mottling
- gentle layered paper feel
- subtle stitched edge suggestions
- quiet support textures behind hero areas or shell backgrounds

Bad texture ideas:
- visible heavy grunge
- scrapbook clutter
- repeated loud patterns
- obvious paper tears everywhere
- faux distressed noise
- noisy craft-store collage

### Texture placement rules

Use texture mainly in:
- page shell background
- hero areas
- empty states
- occasional accent strips
- subtle card backdrops

Do not put strong texture in:
- form fields
- dense reading panels
- export controls
- data-heavy content
- places where text clarity matters most

## 9. Pattern language

Approved patterns:
- stitched/dashed lines
- subtle gingham accents
- sparse polka dot accents
- restrained orchard motifs
- occasional paper layer effects

Pattern rules:
- use one main supporting pattern per area
- avoid stacking multiple strong patterns together
- patterns should live in margins, accents, or empty states more than in content cores

### Gingham use
Allowed in:
- footer strips
- small side accents
- empty states
- subtle panel bands

Not allowed in:
- main reading surfaces
- full-page backgrounds
- dense form zones

### Polka dot use
Allowed in:
- tiny decorative support moments
- empty states
- accent blocks

Not allowed in:
- repeated global use
- high-density decorative fills
- important form or results content

## 10. Signature motifs

The app may reuse these signature motifs carefully:

- moss ribbon headers
- stitched divider lines
- soft layered cards
- subtle paper stack feel
- rounded buttons
- orchard-toned pill badges
- gentle teaching-tool warmth
- small apple/leaf/seed references in moderation

These motifs are part of the brand feel but must remain restrained.

## 11. Shape language

Core shape language:
- rounded
- soft
- comfortable
- layered
- welcoming

Recommended radii:
- major cards: 20px to 24px
- smaller cards: 16px to 20px
- buttons: 16px to 18px
- pills/badges: full pill
- input fields: 14px to 16px

Avoid:
- square corners
- stiff corporate rectangles
- overly sharp segmented controls
- hard-edged dashboard boxes

## 12. Shadows and depth

Shadows should feel:
- soft
- warm
- paper-like
- lifted but light

Suggested shadow style:
- very subtle low blur shadows
- soft vertical lift
- warm rather than cold

Avoid:
- dark heavy shadows
- floating glassmorphism
- dramatic SaaS cards
- harsh layered contrast

Depth should come from:
- paper layers
- borders
- soft shadows
- color separation
- light texture

Not from:
- aggressive drop shadows
- glossy effects
- glass blur
- 3D toy UI

## 13. Layout philosophy

Layout should be:

- calm
- centered
- readable
- teacher-friendly
- guided
- consistent
- spacious without feeling empty

The app should feel like a sequence of well-designed desk papers and planning cards.

### Layout priorities

1. workflow clarity
2. reading comfort
3. teacher trust
4. elegant polish
5. subtle visual delight

### General layout rules

- use one strong primary column feel even when cards are grouped
- avoid dashboard overload
- avoid too many same-weight panels fighting each other
- make the current task obvious
- group related information clearly
- preserve breathing room

## 14. Page shell spec

The page shell should have:

- orchard cream base
- subtle paper/canvas texture
- centered content wrap
- consistent top and bottom padding
- enough width for comfortable planning
- calm negative space around cards

The page shell should feel like the desk or canvas the rest of the paper pieces sit on.

## 15. Hero area spec

Hero areas should:

- welcome the teacher into the step
- establish tone
- explain purpose
- show current context
- feel slightly more decorative than lower cards

Hero areas can include:
- ribbon header
- stitched divider
- serif heading
- gentle support copy
- small side summary card

Hero areas should not:
- become visually heavy banners
- use large illustrations that dominate the workflow
- distract from task completion

## 16. Card system

Cards are the core organizational unit.

Card qualities:
- white or cream-toned surfaces
- warm gray border
- soft shadow
- rounded corners
- comfortable padding
- subtle layer feel

Card types:
- hero cards
- standard workflow cards
- soft support cards
- warning cards
- empty-state cards
- preview cards

### Card hierarchy

Hero cards:
- biggest presence
- most decorative
- introduction and context

Standard workflow cards:
- clear section containers
- most common
- structured and practical

Soft support cards:
- lighter emphasis
- hints, summaries, or side context

Warning cards:
- still warm, not harsh
- use tinted cream/blush/orange tones
- do not feel error-red unless truly critical

Preview cards:
- should look like polished mini teaching artifacts
- a little more personality than form cards

## 17. Buttons

Primary buttons:
- moss/deep orchard emphasis
- white text
- softly elevated
- rounded
- strong but not loud

Secondary buttons:
- cream or pale paper fill
- warm border
- clear text
- quieter hierarchy

Link buttons:
- moss/deep orchard text
- clean and readable
- not underlined unless helpful

Button feeling:
- polished
- tactile
- pressable
- teacher-friendly

Avoid:
- glossy app-store style buttons
- bright blue CTA buttons
- tiny utility links everywhere
- aggressive motion or bounce

## 18. Inputs and form fields

Inputs should feel:
- soft
- calm
- paper-like
- readable
- not clinical

Input rules:
- white or very light paper fill
- warm gray border
- rounded corners
- soft inset feel
- comfortable padding
- consistent sizing

Textareas should:
- feel roomy
- support planning
- not feel cramped
- encourage writing

Labels should:
- be clear and teacher-practical
- slightly bold
- not overly decorative

Help text should:
- explain just enough
- reduce uncertainty
- stay quiet but visible

## 19. Progress stepper

The progress stepper should feel:
- stitched
- guided
- calm
- teacher-supportive

It should clearly show:
- completed
- current
- upcoming

Avoid:
- hard corporate breadcrumb styling
- loud checkmarks
- over-animated progress bars

## 20. Upload workspace styling

The upload area should feel:
- obvious
- modern
- forgiving
- welcoming
- organized

Drop zones should:
- look like elegant paper drop trays
- use soft dashed borders
- show light hover/drag states
- clearly distinguish curriculum vs exemplar

Uploaded item cards should:
- feel tidy
- show file type clearly
- show source role clearly
- preview just enough text
- allow remove easily
- never feel technical or developer-heavy

## 21. Clarification and trust blocks

Clarification blocks are important to the teacher-trust workflow.

They should feel:
- supportive
- thoughtful
- informative
- not scary

These blocks should explain:
- what seems conflicting
- why the generator may pull in two directions
- what the teacher may want to clarify

Tone should be:
- calm
- respectful
- transparent
- non-blaming

## 22. Results page styling

The Results page should feel like a polished teacher workspace, not a debug dump.

It should have:
- a clear lesson identity
- calm export controls
- strong section separation
- polished previews
- readable teacher plan areas
- cleaner interpretation of the generated content

The Results page should communicate:
- what was generated
- what influenced it
- what the teacher can do next

It should not feel:
- like raw JSON with lipstick
- like a developer panel
- like a slide app for kids

## 23. Slide preview deck styling

Slide previews should feel closer to real teaching slides than to plain cards.

They should:
- have stronger visual hierarchy
- look like mini slide canvases
- support teaching use
- feel teacher-led and classroom-usable

For early elementary explicit instruction:
- slide previews should visually support teacher-led teaching
- avoid self-guided navigation mood
- avoid making the deck feel like a kid-facing software menu

## 24. Results content tone

Results wording should be:
- teacher-friendly
- plain-English
- professional
- actionable
- appropriate to grade level

For kindergarten and early elementary:
- do not use self-guided student navigation phrasing
- do not overuse structural jargon
- teacher notes may include cues
- main visible wording should support direct instruction

## 25. Export area styling

Export controls should:
- feel reliable
- look like a polished final step
- not feel technical
- fit the orchard/paper aesthetic

ZIP, PPTX, and DOCX export actions should feel:
- calm
- trustworthy
- high-value
- teacher-ready

## 26. Empty states

Empty states should:
- feel warm
- gently helpful
- not cold or machine-like
- use subtle pattern or motif support

They may include:
- tiny decorative orchard accents
- gentle paper framing
- clear call to action

They should not:
- feel cartoonish
- use emoji clutter
- become more decorative than the real content

## 27. Motion and interaction

Motion should be:
- subtle
- soft
- meaningful
- low-stress

Good motion:
- small hover lifts
- gentle fades
- soft state transitions
- restrained expansion/collapse

Bad motion:
- bouncy toy interactions
- flashy dashboard transitions
- attention-grabbing animation loops
- lots of sliding panels

## 28. Copy tone for UI

UI copy should be:
- calm
- useful
- teacher-smart
- plain
- gently guiding

Avoid:
- overexcited edtech language
- corporate product-speak
- robotic system voice
- childish classroom slogans

The app should sound like:
- a competent instructional partner
- a polished planning tool
- a calm colleague

## 29. Accessibility rules

The design must remain readable and usable.

Requirements:
- sufficient contrast
- legible text size
- decorative texture never interfering with text
- form states clearly visible
- interactive elements obviously interactive
- serif headlines readable, not overly thin
- body text always optimized for sustained reading

## 30. Strict avoid list

Never drift into:

- sterile SaaS dashboard visuals
- generic enterprise admin panel styling
- bright neon educational toy palette
- harsh shadows
- square hard UI boxes
- busy scrapbook clutter
- too many textures at once
- decorative overload
- corporate blue as main identity
- emoji-heavy classroom graphics
- childish cartoon theming
- visually noisy patchwork design
- loud gradient-heavy product design

If something feels:
- corporate
- harsh
- generic
- cluttered
- childish
- overdesigned

it is wrong for this product.

## 31. Design implementation priority

When styling or polishing, prioritize in this order:

1. workflow clarity
2. teacher trust
3. readability
4. visual coherence
5. orchard/storybook warmth
6. scrapbook texture refinement
7. decorative details last

## 32. Decision rule

When there is a conflict between possible visual directions:

Priority order:
1. teacher-first workflow clarity
2. Apple Orchard Storybook direction
3. classy scrapbook texture request
4. current streamlined product structure
5. existing shared orchard UI helpers
6. legacy theme leftovers last

If a choice improves decoration but hurts clarity, clarity wins.
If a choice improves warmth but hurts trust, trust wins.
If a choice feels crafty but messy, it is too far.

## 33. Practical translation of "classy scrapbook with different textures"

This phrase should be interpreted as:

- layered paper feeling
- subtle texture variation between shell, hero, and cards
- stitched and crafted details in moderation
- curation, not clutter
- elegance with warmth
- handmade feeling without mess
- visual richness without noise

It should NOT be interpreted as:

- random scrapbook stickers everywhere
- mixed fonts everywhere
- visible tape, doodles, labels, and craft clutter all at once
- messy collage aesthetics
- overloaded teacher-blog visuals

The correct execution is:
- refined
- soft
- selective
- composed
- warm
- tactile

## 34. Current app application

This full style spec should apply to:

- shared orchard UI tokens and helpers
- Inputs page
- Materials page
- Results page
- upload workspace
- preview cards
- export zone
- empty states
- future teach-mode or slide preview improvements

This spec does not authorize importing old dashboard architecture into the current streamlined wizard app.

## 35. Final one-sentence standard

The Lesson Generator should look and feel like a beautifully organized teacher planning desk made of warm paper, subtle texture, and orchard-toned structure - a classy scrapbook-inspired instructional workspace that is polished, calm, and deeply teacher-first.
